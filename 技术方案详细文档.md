## 1. 项目概述

### 1.1 项目定位

TrendRadar 是一个**轻量级、易部署的热点新闻聚合与 AI 分析工具**，核心目标是：

- **告别无效刷屏**：精准过滤关心的新闻，节省时间
- **AI 智能分析**：深度洞察热点趋势、情感倾向、跨平台关联
- **多渠道推送**：支持 9 大通知渠道，自动格式适配
- **30 秒部署**：GitHub Actions / Docker / 本地部署，零配置起步

### 1.2 技术栈概览

| 分类 | 技术选型 | 说明 |
|------|---------|------|
| **语言** | Python 3.10+ | 简洁高效，生态丰富 |
| **Web 框架** | FastMCP 2.0 | Model Context Protocol 服务器 |
| **数据库** | SQLite 3 | 轻量级嵌入式数据库，无需额外部署 |
| **云存储** | S3 兼容协议 | 支持 R2/OSS/COS/S3，GitHub Actions 云端数据持久化 |
| **AI 接口** | LiteLLM | 统一 100+ AI 提供商接口 |
| **RSS 解析** | feedparser | 支持 RSS 2.0 / Atom 1.0 |
| **HTTP 客户端** | requests + tenacity | 带重试机制的 HTTP 请求 |
| **通知推送** | Webhook + SMTP + API | 支持 9 大主流渠道 |
| **配置管理** | PyYAML + 环境变量 | 灵活的配置覆盖机制 |
| **时区处理** | pytz | 精确的时区转换 |
| **部署方式** | GitHub Actions / Docker / 本地 | 三种部署模式自由切换 |

### 1.3 项目版本

- **核心版本**：v6.0.0

### 2.2 智能分析

#### 2.2.1 关键词统计

**词频分析引擎**（`trendradar/core/analyzer.py`）：
```python
def count_word_frequency(
    data: List[NewsItem],
    word_groups: List[Dict],
    filter_words: Set[str],
    global_filters: Set[str]
) -> Tuple[Dict, Dict, List]:
    """
    输入：新闻数据 + 词组配置 + 过滤词
    输出：
      - keyword_stats: {关键词: {新闻列表, 统计信息}}
```

**支持语法**：
- **普通词匹配**：`华为`、`特斯拉`
- **正则表达式**：`/\bAI\b/`、`/鸿蒙|HarmonyOS/`
- **必须词**：`+任正非`（标题必须包含）
- **过滤词**：`!广告`（标题包含则跳过）
- **显示别名**：`/pattern/ => 显示名称`
- **组别名**：`[AI技术]` 作为分组标题
- **最大显示数**：`@10` 限制该组最多显示 10 条

#### 2.2.2 权重计算

**权重公式**：
```python
综合权重 = 排名权重 + 频次权重 + 热度权重

排名权重：越靠前权重越高（对数衰减）
频次权重：出现次数越多权重越高
热度权重：平台热度值归一化

def calculate_news_weight(
    rank: int,          # 排名（1-50）
    frequency: int,     # 出现频次
    hotness: float,     # 热度值
    rank_weight: float = 0.4,
    freq_weight: float = 0.3,
    hot_weight: float = 0.3
) -> float:
    rank_score = (51 - rank) / 50
    freq_score = min(frequency / 10, 1.0)
    hot_score = min(hotness / 1000000, 1.0)

    return (
        rank_score * rank_weight +
        freq_score * freq_weight +
        hot_score * hot_weight
    )
```

#### 2.2.3 趋势识别

**热度趋势箭头**：
- 🔺 上升：本次权重 > 上次权重
- 🔻 下降：本次权重 < 上次权重
- ➖ 持平：权重无变化或首次出现

**生命周期分析**（MCP 工具）：
- 昙花一现：热度持续 < 6 小时
- 短期热点：6-24 小时
- 持续热点：24-72 小时
- 长期话题：> 72 小时

### 2.3 AI 深度分析

#### 2.3.1 分析维度（6 大板块）

**1. 核心热点与舆情态势**
- 识别当前最受关注的核心议题
- 分析跨平台热度分布
- 评估舆论整体态势

**2. 舆论风向与争议**
- 情感倾向识别（正面/负面/中性）
- 争议性话题检测
- 舆论分化程度评估

**3. 异动与弱信号**
- 突然爆火的话题（异常检测）
- 潜在热点预警
- 非主流但值得关注的信号

**4. RSS 深度洞察**
- 技术社区热点
- 深度内容推荐
- 行业趋势总结

**5. 研判与策略建议**
- 未来趋势预测
- 关注重点建议
- 风险提示

**6. 独立源点速览**（v6.0.0 新增）
- 对每个独立展示的平台/RSS 源生成核心概括
- 包含排名/时间/轨迹数据
- 支持热榜和 RSS 源

## 4. 技术实现细节

### 4.1 核心算法

#### 4.1.1 增量检测算法

```python
# trendradar/core/data.py
def detect_latest_new_titles_from_storage(
    storage_backend: StorageBackend,
    date_str: str
) -> Tuple[List[NewsItem], Dict[str, str], Dict]:
    """
    检测当天最新批次的新增标题

    算法步骤：
    1. 获取当天所有批次数据
    2. 取最新批次的标题集合 A
    3. 取历史批次的标题集合 B
    4. 新增标题 = A - B（集合差运算）
    5. 标记新增标题（添加 🆕 emoji）
    """
    # 获取所有批次
    all_batches = storage_backend.get_today_all_data(date_str)

    # 最新批次
    latest_batch = all_batches[-1]
    latest_titles = {item.title for item in latest_batch}

    # 历史批次
    history_titles = set()
    for batch in all_batches[:-1]:
        history_titles.update({item.title for item in batch})

    # 新增标题
    new_titles = latest_titles - history_titles

    # 标记新增
    for item in latest_batch:
        if item.title in new_titles:
            item.title = f"🆕 {item.title}"

    return latest_batch, build_platform_map(latest_batch), {
        "new_count": len(new_titles),
        "new_titles": list(new_titles)
    }
```

#### 4.1.2 URL 标准化算法

```python
# trendradar/utils/url.py
def standardize_url(url: str) -> str:
    """
    URL 标准化，移除动态参数

    处理的参数：
    - band_rank (微博排名)
    - utm_* (追踪参数)
    - timestamp (时间戳)
    - session_id (会话 ID)
    """
    from urllib.parse import urlparse, parse_qs, urlencode, urlunparse

    parsed = urlparse(url)
    query_params = parse_qs(parsed.query)

    # 移除动态参数
    DYNAMIC_PARAMS = {
        'band_rank', 'utm_source', 'utm_medium',
        'utm_campaign', 'timestamp', 'session_id'
    }

    filtered_params = {
        k: v for k, v in query_params.items()
        if k not in DYNAMIC_PARAMS
    }

    new_query = urlencode(filtered_params, doseq=True)

    return urlunparse((
        parsed.scheme, parsed.netloc, parsed.path,
        parsed.params, new_query, parsed.fragment
    ))
```
