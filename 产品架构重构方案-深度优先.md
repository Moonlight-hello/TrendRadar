# TrendRadar 产品架构重构方案 - 深度优先

> **核心理念转变**: 从"广度聚合"到"垂直深挖"
> **目标用户**: 专业投资者、产业分析师、行业从业者
> **价值主张**: 对少数品种建立完整的数据体系，支撑投资决策

---

## 📊 一、现有架构 vs 新架构对比

## 二、核心理念：单品深度 > 多品广度

### 2.1 为什么要"深度优先"？

```
┌────────────────────────────────────────────────────┐
│           广度优先 vs 深度优先                      │
├────────────────────────────────────────────────────┤
│                                                     │
│  广度优先（现有TrendRadar）                        │
│  ├─ 适用场景: 泛资讯消费                           │
│  ├─ 用户特征: 兴趣广泛，浅层了解                   │
│  ├─ 决策需求: 无（只是看看热闹）                   │
│  └─ 配置逻辑: 想看什么加什么关键词                 │
│                                                     │
│  深度优先（原料库存监控）                          │
│  ├─ 适用场景: 专业投资、产业决策                   │
│  ├─ 用户特征: 专注少数品种，深度研究               │
│  ├─ 决策需求: 高（买卖决策、风险管理）             │
│  └─ 配置逻辑: 开通品种=建立完整数据体系            │
│                                                     │
└────────────────────────────────────────────────────┘

关键洞察：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
"用户只会关注自己认知范围内的产品"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

例如：
  铜贸易商     → 只关注铜，不关心铝锌镍
  铝冶炼厂     → 只关注铝和氧化铝
  大宗商品基金 → 关注3-5个主力品种

因此：
  ✗ 配置50个关键词，每个浅尝辄止 → 无价值
  ✓ 配置3个品种，每个深度建模 → 高价值
```

### 2.2 什么是"深度"？

```
┌────────────────────────────────────────────────────┐
│              单品深度的6个维度                      │
├────────────────────────────────────────────────────┤
│                                                     │
│  1️⃣ 数据完整性                                     │
│  ├─ 供给侧: 产量、进口、矿山、冶炼                 │
│  ├─ 需求侧: 消费、出口、下游行业                   │
│  ├─ 库存: 交易所、保税区、社会                     │
│  ├─ 价格: 现货、期货、升贴水                       │
│  └─ 宏观: PMI、GDP、固投、利率                     │
│                                                     │
│  2️⃣ 时间连续性                                     │
│  ├─ 历史数据: 至少3年                              │
│  ├─ 更新频率: 日度/周度/月度                       │
│  └─ 实时追踪: 价格、库存、新闻                     │
│                                                     │
│  3️⃣ 分析模型                                       │
│  ├─ 供需平衡模型                                   │
│  ├─ 库存天数模型                                   │
│  ├─ 价格驱动因素模型                               │
│  └─ 异常检测模型                                   │
│                                                     │
│  4️⃣ 预警体系                                       │
│  ├─ 供应风险预警                                   │
│  ├─ 需求风险预警                                   │
│  ├─ 库存风险预警                                   │
│  └─ 异常行为预警（囤货、逼仓）                     │
│                                                     │
│  5️⃣ 新闻与数据关联                                 │
│  ├─ 不是孤立看新闻                                 │
│  ├─ 不是孤立看数据                                 │
│  └─ 而是自动关联分析                               │
│                                                     │
│  6️⃣ AI深度洞察                                     │
│  ├─ 基于完整数据的AI分析                           │
│  ├─ 价格驱动因素拆解                               │
│  ├─ 供需态势研判                                   │
│  └─ 投资建议与风险提示                             │
│                                                     │
└────────────────────────────────────────────────────┘

对比示例：

浅层（现有）:
  "今日铜相关新闻3条，AI分析：铜价上涨受需求拉动"

深度（新架构）:
  "铜价上涨5.2%
   【供给】产量-2.3%，进口+15.8%，总供给127.7万吨
   【需求】消费+3.5%，总需求133.5万吨
   【平衡】短缺4.8万吨（-5.7%）⚠️
   【库存】22.3天，低于安全水平 ⚠️
   【异常】检测到囤货风险：进口激增+价格上涨+库存不降
   【驱动】60%基本面+20%宏观+20%投机
   【建议】谨慎偏多，目标77,000-78,000元/吨，止损74,000"

→ 深度版本可以直接用于交易决策！
```

## 三、新架构设计：品种卡片系统

### 3.1 配置文件重构

**从"关键词配置"到"品种卡片"**

```yaml
# config/products.yaml（新）
# ═══════════════════════════════════════════════════════════
#                    产品品种配置文件
#     理念：开通一个品种 = 建立完整的数据监控体系
# ═══════════════════════════════════════════════════════════

products:
  # ────────────────────────────────────────────────────────
  # 品种1: 铜（COPPER）
  # ────────────────────────────────────────────────────────
  copper:
    # 基础信息
    name: "铜"
    name_en: "Copper"
    symbol: "Cu"
    enabled: true

    # 关键词（用于新闻抓取）
    keywords:
      primary: ["铜", "copper", "Cu"]
      alias: ["电解铜", "阴极铜", "精炼铜"]
      exclude: ["铜钱", "铜像"]  # 排除词

    # 数据源配置
    data_sources:
      # 供给侧
      supply:
        - type: "production"
          name: "国内产量"
          source: "china_nia"  # 中国有色金属工业协会
          frequency: "monthly"
          enabled: true

        - type: "import"
          name: "进口量"
          source: "china_customs"  # 海关总署
          frequency: "monthly"
          enabled: true

        - type: "mine_production"
          name: "矿山产量"
          source: "icsg"  # 国际铜研究组织
          frequency: "monthly"
          enabled: false  # 可选数据源

      # 需求侧
      demand:
        - type: "consumption"
          name: "表观消费"
          source: "china_nia"
          frequency: "monthly"
          enabled: true

        - type: "export"
          name: "出口量"
          source: "china_customs"
          frequency: "monthly"
          enabled: true

        - type: "downstream"
          name: "下游需求"
          sectors:
            - name: "电力"
              weight: 0.45  # 占比45%
            - name: "家电"
              weight: 0.15
            - name: "汽车"
              weight: 0.12
            - name: "建筑"
              weight: 0.28
          enabled: true

      # 库存
      inventory:
        - type: "exchange"
          name: "交易所库存"
          sources:
            - name: "LME"
              url: "https://www.lme.com/Metals/Non-ferrous/LME-Copper/Stocks"
              frequency: "daily"
            - name: "SHFE"
              url: "http://www.shfe.com.cn/statements/dataview.html"
              frequency: "daily"
            - name: "COMEX"
              url: "https://www.cmegroup.com/delivery_reports/"
              frequency: "daily"
              enabled: false

        - type: "bonded"
          name: "保税区库存"
          source: "smm"
          frequency: "weekly"
          enabled: true

        - type: "social"
          name: "社会库存"
          source: "smm"
          frequency: "weekly"
          enabled: true

      # 价格
      price:
        - type: "spot"
          name: "现货价格"
          sources:
            - name: "上海有色网"
              source: "smm"
              frequency: "daily"
              realtime: true
            - name: "长江有色"
              source: "ccmn"
              frequency: "daily"
              enabled: false

        - type: "future"
          name: "期货价格"
          sources:
            - name: "SHFE主力合约"
              source: "shfe"
              frequency: "realtime"
            - name: "LME三月期"
              source: "lme"
              frequency: "realtime"

        - type: "premium"
          name: "升贴水"
          calculation: "spot - future"
          frequency: "daily"

      # 宏观数据
      macro:
        - type: "pmi"
          name: "PMI指数"
          source: "nbs"
          frequency: "monthly"

        - type: "fixed_investment"
          name: "固定资产投资"
          source: "nbs"
          frequency: "monthly"

        - type: "industrial_added_value"
          name: "工业增加值"
          source: "nbs"
          frequency: "monthly"

    # 分析模型配置
    analysis_models:
      # 供需平衡模型
      supply_demand:
        enabled: true
        calculation: "supply - demand"
```

## 六、关键问题与解答

### Q1: 是否完全抛弃原有的广度模式？

**A**: 不，兼容共存。

```yaml
# config/config.yaml
mode: "hybrid"  # classic / depth / hybrid

# classic: 原有的广度模式（关键词匹配）
# depth: 新的深度模式（品种监控）
# hybrid: 混合模式（两者并存）

classic_mode:
  enabled: true
  config_file: "frequency_words.txt"

depth_mode:
  enabled: true
  config_file: "products.yaml"

# 推送时分别处理
notification:
  classic_section: true  # 显示关键词匹配的新闻
  depth_section: true    # 显示品种深度分析
```

### Q2: 用户如何选择使用哪种模式？

**A**: 根据使用场景。

```
广度模式（Classic）:
  适用场景: 泛资讯消费、多领域关注
  配置方式: frequency_words.txt
  推送内容: 热榜新闻 + AI浅层分析

深度模式（Depth）:
  适用场景: 专业投资、产业决策
  配置方式: products.yaml
  推送内容: 品种日报（新闻+数据+分析+预警）

混合模式（Hybrid）:
  适用场景: 既要泛资讯，又要深度监控核心品种
  配置方式: 两者都配置
  推送内容: 分section展示
```

### Q3: 数据源获取有哪些挑战？

**A**: 主要挑战和应对策略。

```
挑战1: 数据源需要登录/付费
  应对:
    ├─ 优先使用公开免费数据源
    ├─ 用户自行购买数据服务（如Wind、SMM）
    └─ 提供数据源适配接口，用户可接入自己的数据

挑战2: 数据格式不统一
  应对:
    └─ 设计数据源适配器基类，规范化输出

挑战3: 数据更新频率不一致
  应对:
    └─ 按数据类型设置不同的更新策略
       ├─ 实时数据（价格）: 5分钟
       ├─ 日度数据（库存）: 1小时
       ├─ 月度数据（进出口）: 1天

挑战4: 反爬虫限制
  应对:
    ├─ 设置合理的请求间隔
    ├─ 使用官方API（优先）
    ├─ 代理轮换
    └─ 降低爬取频率
```

### Q4: 如何保证分析的准确性？

**A**: 多层验证。

```
数据质量保证:
  ├─ 数据验证（范围检查、异常值检测）
  ├─ 多数据源交叉验证
  └─ 历史数据一致性检查

模型验证:
  ├─ 回测验证（历史数据验证模型准确性）
  ├─ 专家规则（行业经验规则）
  └─ 持续优化（根据反馈调整）

AI分析:
  ├─ 基于完整数据（不是臆测）
  ├─ 提示词工程（精确的提示词）
  └─ 人工审核（关键决策需人工确认）
```

```

  ✓ 深度分析，支撑决策
  ✓ 异常预警，规避风险

vs 数据终端（Wind/Bloomberg）:
  ✓ 开源免费，可定制
  ✓ 自动化分析，无需人工
  ✓ 新闻数据关联，全面洞察

vs AI助手（ChatGPT）:
  ✓ 持续监控，不是一次性查询
  ✓ 数据实时更新，不是静态知识
  ✓ 主动推送，不需要反复询问
```
